# Builder 패턴
**Builder 패턴**은 **복잡한 객체를 단계적으로 만들 수 있게 하는 방법**이다.  
특히 **필드가 많거나 선택 옵션이 많은 객체**를 깔끔하게 생성하는 데 유용하다.

---

## 장점
| 장점          | 설명                        |
| ----------- | ------------------------- |
| 읽기 쉬움       | 어떤 값이 어떤 필드에 들어가는지 명확함    |
| 선택적 값 처리 쉬움 | 필요한 값만 설정하면 됨             |
| 불변 객체 만들기 쉬움 | `build()` 후 값 변경 없이 사용 가능 |

## 단점
| 단점          | 설명                        |
| ----------- | ------------------------- |
| 코드 양이 늘어남   | Builder 클래스를 추가로 작성해야 함   |
| 간단한 객체에는 과함 | 필드가 1~2개뿐이라면 일반 생성자가 더 간단 |

## 요약
Builder 패턴은 "필드가 많은 객체"를 만들 때 특히 유용하다.
가독성이 좋아지고 실수를 줄일 수 있지만, 객체 구조가 단순할 때는 굳이 사용할 필요는 없다.
